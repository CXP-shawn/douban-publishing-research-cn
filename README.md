# 豆瓣自动化发布通道调研报告

> **任务**：在豆瓣官方 OAuth 未开放的情况下，调研"自动发布读书笔记、书评、日记、广播"的现实方案。
> **结论**：没有任何银弹。所有路线都依赖账号 Cookie 或 App 抓包，且都承担被风控/封号风险。最靠谱做法是**浏览器自动化（Playwright）+ storage_state 复用登录态**，在 Twin 中用 `computer_use_agent` 兜底验证码。
> **调研时间**：2026-04-21（UTC）。

## ⚡ 执行摘要（80 字）

豆瓣官方 OAuth 自 2014 起停止新应用审核，2019 年后事实停摆；`developers.douban.com` 已不可访问，`api.douban.com` 仅向商务合作方开放。frodo 私有 API 签名可逆向但**无任何公开仓库实现"写入"**。推荐路线：**Playwright + Cookie 登录态 + 人工兜底验证码**，每天限 2–5 条，可在 Twin 内用 `computer_use_agent` 实现。

## 🏁 终极建议（一句话）

> **选方案 #1：Playwright + 复用 storage_state Cookie，用 Twin 的 `computer_use_agent` 做首次登录与偶发极验兜底，日发布量控制在 ≤5 条以降低风控命中率。**

详情见 [REPORT.md 第 7 章 · 最终推荐](./REPORT.md#7-最终推荐如果我是用户我会选哪个方案)。
